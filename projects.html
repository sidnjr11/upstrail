Project Clearmesh [cite: 67]

Automated Data Ingestion & Semantic Mapping for Supply Chain Systems

Semantic Embedding Python OpenAI API Automated Mapping

Objective

[cite_start]

The Clearmesh system is designed to automate the ingestion of "messy," unstructured client data files (CSV format) and map them to a canonical, structured schema (Advanced Planning and Scheduling system). [cite: 68]

[cite_start]

It replaces brittle, rules-based mapping (like Regex) with Semantic Matching, allowing the system to map a column named cust_id_x to Customer ID simply because they behave similarly in vector space. [cite: 145, 146]

Core Mechanism

[cite_start]

The system utilizes Semantic Embedding (via OpenAI's text-embedding-3-small model) and Cosine Similarity to match incoming data against a known "Gold Standard" training dataset. [cite: 69]

System Architecture

[cite_start]

The workflow operates in two distinct phases: [cite: 70]

1. Preprocessing (Training Phase)

[cite_start]

This phase creates a knowledge base of embeddings for known file types (e.g., SAP Tables) and their expected columns. [cite: 78]

    [cite_start]
  • Data Ingestion: Reads raw training data and groups it by table type (e.g., extracting metadata like Expected Data Type and Description). [cite: 80, 81]
    • [cite_start]
  • Embedding Generation: Generates text summaries for file types and columns, then converts them into vector embeddings stored in CSVs. [cite: 85, 86]

2. Inference (Runtime Phase)

[cite_start]

This phase processes incoming files to identify their type and map their columns. [cite: 92]

    [cite_start]
  • File Type Detection: Generates a "fingerprint" of the incoming file (using header names and first-row data types) and performs a vector search to find the best matching file type (e.g., "Order File" vs "Material File"). [cite: 108, 109]
    • [cite_start]
  • Column Mapping: Once the file type is identified, the system generates descriptions for every column (including sample data and key constraints) and matches them to the canonical schema using Cosine Similarity. [cite: 120, 128]
